# 产品需求文档（PRD）- 今天吃什么？

## 一、目标用户

| 用户画像 | 描述 |
|---------|------|
| **选择困难症用户** | 每天面临"吃什么"的选择困难，需要快速决策支持 |
| **饮食偏好明确用户** | 有明确的忌口、口味偏好，希望推荐更精准 |
| **追求效率用户** | 希望通过简单操作快速获得推荐，减少决策时间 |
| **数据驱动用户** | 关注自己的饮食偏好统计，希望看到数据反馈 |

---

## 二、核心痛点

1. **选择困难** - 每天面临"吃什么"的决策困境，浪费时间
2. **推荐不精准** - 随机推荐不考虑用户偏好、忌口、时段等因素
3. **缺乏反馈机制** - 无法记录用户对推荐的反馈，无法优化推荐算法

---

## 三、核心功能

### 功能1：智能推荐系统
**功能描述**：基于用户偏好、时段、忌口的加权随机推荐

**核心流程**：
1. 用户输入忌口食材（可选）
2. 选择分类筛选（可选）
3. 点击"随机推荐"按钮
4. 系统根据时段、点赞权重、不喜欢权重计算推荐结果
5. 展示推荐菜品及分类标签

**时段智能规则**：
- 早餐（05:00-10:00）：优先推荐主食、汤类
- 午餐（10:00-14:00）：全品类推荐
- 下午茶（14:00-17:00）：优先推荐甜点、主食
- 晚餐（17:00-22:00）：全品类推荐
- 夜宵（22:00-05:00）：优先推荐主食、汤类

**加权算法**：
```
推荐权重 = 基础权重(1) + 点赞数 × 0.5 - 不喜欢数 × 0.8
```

---

### 功能2：偏好反馈系统
**功能描述**：用户对推荐结果进行点赞/不喜欢反馈，优化后续推荐

**点赞功能**：
- 点击"点赞"按钮，该菜品点赞数+1
- 点赞数增加该菜品推荐权重50%
- 实时更新数据统计面板

**不喜欢功能**：
- 点击"不喜欢，换掉"按钮，该菜品不喜欢数+1
- 不喜欢数降低该菜品推荐权重80%
- 自动重新推荐下一道菜

**数据持久化**：
- 所有反馈数据存储在浏览器 localStorage
- 跨会话保持用户偏好数据

---

### 功能3：数据统计面板
**功能描述**：可视化展示用户饮食偏好数据

**统计模块**：

| 模块 | 内容 |
|-------|------|
| **TOP5 排行** | 点赞最多的5道菜，金银铜牌标识 |
| **偏好分析** | 各分类（荤菜/素菜/汤类/主食/甜点）点赞分布横向条形图 |
| **统计摘要** | 菜品总数、总点赞数、总踩数 |

**实时更新**：
- 点赞/不喜欢后自动刷新统计
- 添加/删除菜品后自动更新
- 分类筛选不影响统计面板（始终显示全局数据）

## 四、数据模型

### 4.1 菜品数据（Dishes）
```json
{
  "name": "红烧肉",        // 菜品名称（必填）
  "category": "荤菜",     // 分类：荤菜/素菜/汤类/主食/甜点（必填）
  "breakfast": true        // 是否为早餐专属菜品（可选）
}
```

### 4.2 点赞数据（DishLikes）
```json
{
  "红烧肉": 5,    // 菜品名称: 点赞数
  "宫保鸡丁": 3,
  "麻婆豆腐": 2
}
```

### 4.3 不喜欢数据（DishDislikes）
```json
{
  "香菜": 2,      // 菜品名称: 不喜欢数
  "鱼香肉丝": 1
}
```

### 4.4 时段配置（TimePeriods）
```json
{
  "breakfast": {
    "name": "早餐",
    "icon": "🌅",
    "start": 5,
    "end": 10,
    "categories": ["主食", "汤类"]
  }
}
```
